# 🔍 UX Research Prompt: Fitness App User Experience Best Practices

## 📋 調査目的

フィットネス・トレーニング記録アプリにおける**最高のユーザー体験（UX）**を調査し、以下の課題を解決する：

### 🚨 現在の問題
- **App Store評価**: 3.0 / 5.0（低評価）
- **ユーザーコメント**: 「使わないと、たくさんのことを見逃してしまいます」
- **課題**: 機能が隠れている、初回体験が不十分、価値が伝わらない

---

## 🎯 調査すべき重点項目

### 1. オンボーディング（初回体験）
**調査内容**:
- 成功しているフィットネスアプリの初回体験フロー
- ユーザーに価値を伝える最適な方法
- スキップ可能 vs 必須チュートリアルの比較
- 最適なオンボーディング画面数（3枚？5枚？）

**参考アプリ**:
- Strong (筋トレ記録)
- MyFitnessPal (フィットネス総合)
- Nike Training Club (トレーニングガイド)
- Strava (ランニング・サイクリング)
- Fitbod (AIトレーニング)

**質問**:
1. トップフィットネスアプリのオンボーディングフローは？
2. ユーザーリテンション率を高めるオンボーディングの要素は？
3. 初回起動後、何%のユーザーが2回目以降使用するか？
4. スキップ率が低いオンボーディングのデザインパターンは？

---

### 2. ホーム画面の設計
**調査内容**:
- 主要機能へのアクセスしやすさ
- クイックアクションの配置
- 視覚的階層（何を目立たせるべきか）
- ユーザーの期待値とのマッチング

**質問**:
1. フィットネスアプリのホーム画面で「最初に見るべき情報」は何か？
2. クイックアクションボタンの最適な配置は？（上部？中央？FAB？）
3. 主要3機能を目立たせる最適なUI配置は？
4. ユーザーが最も頻繁に使う機能TOP3は？

---

### 3. 機能発見可能性（Discoverability）
**調査内容**:
- 隠れた機能をユーザーに気づかせる方法
- プログレッシブディスクロージャー（段階的開示）
- コンテクスチュアルヒント（文脈に応じたヒント）
- 空状態（Empty State）の活用

**質問**:
1. 「見逃される機能」を防ぐベストプラクティスは？
2. ツールチップ、バッジ、ポップアップの使い分けは？
3. ユーザーを邪魔せずに機能を伝える方法は？
4. 初回ユーザー vs リピーターで見せ方を変えるべきか？

---

### 4. AI機能の訴求
**調査内容**:
- AIコーチ、AI予測などの「AI機能」の訴求方法
- ユーザーがAI機能に期待すること
- AI機能の価値を伝える視覚表現

**質問**:
1. フィットネスアプリでAI機能を前面に出している成功例は？
2. ユーザーが「AI」に期待する体験は？（自動化？パーソナライズ？）
3. AIコーチ機能のアイコン・色・文言の最適解は？
4. AI機能の「すごさ」を一瞬で伝えるキャッチコピーは？

---

### 5. モチベーション維持
**調査内容**:
- 成長可視化（グラフ、チャート）
- ゲーミフィケーション要素
- 達成感を与えるUI/UX
- 継続利用を促す通知戦略

**質問**:
1. ユーザーのモチベーションを維持するトップ5の方法は？
2. 成長予測機能はどう見せるべきか？（グラフ？数値？ビジュアル？）
3. トレーニング記録後の「達成感演出」のベストプラクティスは？
4. プッシュ通知の最適な頻度とタイミングは？

---

### 6. データ入力の簡易化
**調査内容**:
- 記録の手間を最小化する方法
- 前回データの活用
- ワンタップ記録
- スマートデフォルト値

**質問**:
1. トレーニング記録で「面倒くささ」を減らすベストプラクティスは？
2. 「前回と同じ」をワンタップで入力できるUI設計は？
3. AIが自動入力すべき項目は何か？
4. 音声入力、カメラ入力など非タッチ入力の有効性は？

---

### 7. 視覚デザイン
**調査内容**:
- フィットネスアプリの色彩設計
- アイコンの選択
- タイポグラフィ
- ダークモード対応

**質問**:

2. 「筋トレ」「有酸素」を視覚的に区別する最良の方法は？
3. 数値データを読みやすく表示する最適なフォントサイズは？
4. ダークモードでのトレーニング記録画面のベストプラクティスは？

---

### 8. ユーザーフィードバック・レビュー改善
**調査内容**:
- 低評価レビューへの対処法
- レビュー依頼のタイミング
- ユーザーの不満を早期発見する方法

**質問**:
1. App Store評価を3.0から4.5に上げた事例は？
2. レビュー依頼の最適なタイミングは？（初回？10回目？達成時？）
3. 「見逃している機能」をユーザーに気づかせた成功事例は？
4. 低評価ユーザーをリテンションする方法は？

## 🔬 調査方法

### A. 競合アプリ分析
以下のトップフィットネスアプリをダウンロードして実際に使用分析：
- **Strong**: 筋トレ記録No.1
- **Fitbod**: AI自動メニュー生成
- **JEFIT**: 豊富な種目データベース
- **Nike Training Club**: ガイド付きトレーニング
- **MyFitnessPal**: 総合フィットネス管理
- **Strava**: ソーシャル機能充実

**分析項目**:
1. 初回起動から最初のトレーニング記録までのステップ数
2. ホーム画面の情報配置
3. 主要機能へのアクセス方法（タップ数）
4. オンボーディングの有無と内容
5. AI機能の訴求方法

---

### B. UX研究論文・記事
以下のソースから最新のベストプラクティスを調査：
- Nielsen Norman Group (UX専門機関)
- Material Design Guidelines (Google)
- Human Interface Guidelines (Apple)
- Baymard Institute (UXリサーチ)
- UX Collective (Medium)

**キーワード検索**:
```
- "fitness app onboarding best practices"
- "mobile app discoverability patterns"
- "AI feature UX design"
- "gamification fitness motivation"
- "progressive disclosure mobile apps"
- "empty state design fitness"
```

---

### C. ユーザー行動データ分析
App Store / Google Playのレビュー分析：
- 高評価（★4-5）アプリのレビュー共通点
- 低評価（★1-2）アプリの共通不満点
- 「見逃した」「分かりにくい」などのキーワード出現率
